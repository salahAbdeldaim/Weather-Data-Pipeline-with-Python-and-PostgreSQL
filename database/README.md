# 🗄️ Database Layer & Connection Guide | طبقة وقاعدة البيانات وكيفية الاتصال بها

هذا المجلد (`database/`) مخصص لكل ما يتعلق بقاعدة بيانات المشروع **PostgreSQL**، ويحتوي على مخطط الجداول (`schema.sql`) والاستعلامات التحليلية واختبار الأداء (`queries.sql`)، بالإضافة إلى دليل الاتصال بقاعدة البيانات مباشرة من جهازك الشخصي (اللاب توب) أثناء المناقشة والتقييم.

---

## 📂 محتويات المجلد (`Contents`)
- **`schema.sql`**: ملف تعريف هيكل قاعدة البيانات (DDL)، يتضمن جدول `weather_readings` الرئيسي والفهارس (`Indexes`) لتحسين سرعة الاستعلامات، بالإضافة إلى الـ `Views` الجاهزة.
- **`queries.sql`**: مجموعة استعلامات جاهزة (`SELECT`) لعرض أحدث البيانات، مقارنة درجات الحرارة بين المدن والمحافظات، والتحقق من سلامة البيانات أثناء العرض التقديمي.

---

## 🔌 كيفية الاتصال بقاعدة البيانات من اللاب توب (`Connecting from Laptop`)

قاعدة البيانات تعمل داخل حاوية دوكر (`Docker Container`) ومربوطة بالمنفذ (`Port 5432`) على جهازك الشخصي، مما يعني أنه يمكنك فتحها والتعامل معها بسهولة لأي غرض أثناء المناقشة باستخدام أي برنامج إدارة قواعد بيانات.

### 🔑 بيانات الاتصال (`Connection Parameters`)
* **Host / Server:** `localhost` أو `127.0.0.1`
* **Port:** `5432`
* **Database Name:** `weather_db` *(أو القيمة المحددة في ملف `.env`)*
* **Username:** `weather_user` *(أو القيمة المحددة في ملف `.env`)*
* **Password:** `weather_pass_2026` *(أو القيمة المحددة في ملف `.env`)*

---

## ❓ هل يمكن الاتصال من خلال برنامج SQL Server Management Studio (SSMS)؟

### 💡 الإجابة باختصار:
برنامج **SSMS** مخصص بشكل أساسي وحصري من مايكروسوفت لإدارة قواعد بيانات **Microsoft SQL Server** فقط، ولا يدعم الاتصال المباشر بقواعد بيانات **PostgreSQL** بشكل افتراضي.

---

## 🏆 ما هي أفضل وأسهل الطرق لفتح الداتا بيز أثناء المناقشة؟ (الطرق الموصى بها)

### 1️⃣ الطريقة الأولى (الأسهل والأسرع - مباشرة من داخل VS Code أو Cursor):
بما أنك تفتح المشروع حالياً داخل المحرر (**Visual Studio Code** أو **Cursor**)، يمكنك التفاعل مع قاعدة البيانات وبناء الاستعلامات دون الحاجة لتثبيت أي برنامج خارجي:
1. افتح متجر الإضافات (`Extensions` بالضغط على `Ctrl+Shift+X`).
2. ابحث وثبّت إضافة **SQLTools** (ومعها تعريف **PostgreSQL/Cockroach Driver**) أو إضافة **Database Client**.
3. اضغط على أيقونة الإضافة، ثم اختر **Add Connection** وحدد **PostgreSQL**.
4. أدخل البيانات المذكورة أعلاه (`localhost`, `5432`, `weather_user`, `weather_pass_2026`, `weather_db`).
5. اضغط **Save & Connect**، ويمكنك الآن تصفح جدول `weather_readings` وتشغيل استعلامات `queries.sql` بضغطة زر!

---

### 2️⃣ الطريقة الثانية (البرنامج الاحترافي الأفضل عالمياً - DBeaver Community):
إذا كنت ترغب في واجهة رسومية منفصلة واحترافية تشبه **SSMS** ولكنها تدعم جميع أنواع قواعد البيانات (بما فيها PostgreSQL):
1. قم بتحميل وتثبيت النسخة المجانية من برنامج **[DBeaver Community Edition](https://dbeaver.io/download/)**.
2. افتح البرنامج واضغط على أيقونة فيشي التوصيل **New Database Connection**.
3. اختر **PostgreSQL** ثم اضغط **Next**.
4. في خانة `Host` اكتب `localhost`، والـ `Port` اكتب `5432`.
5. في خانة `Database` اكتب `weather_db`، وفي `Username` اكتب `weather_user`، وفي `Password` اكتب `weather_pass_2026`.
6. اضغط **Test Connection** ثم **Finish**.
7. ستتمكن من عرض الجداول وتصفح البيانات على هيئة جدول ببيانات حية مباشرة.

---

### 3️⃣ الطريقة الثالثة (برنامج Azure Data Studio من مايكروسوفت):
إذا كنت تحب شكل واجهة **SSMS** التابعة لمايكروسوفت، فيمكنك استخدام البديل الحديث منها وهو **[Azure Data Studio](https://azure.microsoft.com/en-us/products/data-studio)**:
1. قم بتحميل وتثبيت **Azure Data Studio**.
2. من متجر الإضافات داخل البرنامج، ثبّت إضافة **PostgreSQL extension**.
3. أضف اتصالاً جديداً (`New Connection`) وأدخل نفس بيانات الاتصال (`localhost:5432`).

---

## 🚀 كيفية إعادة تهيئة جدول قاعدة البيانات من الصفر (إذا لزم الأمر)
في حال أردت تنظيف قاعدة البيانات أو إعادة إنشائها يدوياً من موجة الأوامر:
```bash
docker exec -it weather_postgres psql -U weather_user -d weather_db -f /var/lib/postgresql/data/schema.sql
```
أو ببساطة تشغيل بايبلين استخراج الطقس الذي يقوم بإنشاء الجدول وتحديثه تلقائياً:
```bash
docker compose start weather-etl
```
