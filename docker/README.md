# 🐳 Docker & DevOps Layer | طبقة إعدادات وحاويات الدوكر

هذا المجلد (`docker/`) مخصص لإدارة حاويات المشروع وإعدادات التشغيل السريع. يتيح لأعضاء الفريق والمقيّمين (`Evaluators`) فحص إعدادات الدوكر وتشغيل أو إيقاف كافة الخدمات بسهولة تامة.

---

## 📦 الخدمات المشغّلة داخل الدوكر (`Docker Services`)

يتكون النظام من ثلاث حاويات رئيسية معرّفة داخل ملف `docker-compose.yml`:

1. **`weather_postgres` (`PostgreSQL Database`)**:
   - حاوية قاعدة البيانات الرئيسية بمحرك `PostgreSQL 15`.
   - مربوطة بالمنفذ (`Port 5432`) ليتمكن المطور أو المقيّم من فتحها والتعامل معها من داخل اللاب توب مباشرة (عبر VS Code أو DBeaver).
   - تقوم تلقائياً بتهيئة الجداول من خلال ربط ملف `database/schema.sql` عند أول تشغيل للحاوية.

2. **`weather_etl_pipeline` (`Python ETL Layer`)**:
   - حاوية بايثون تقوم بتشغيل كود الاستخراج (`api/main.py` و `api/loader.py`).
   - تسحب بيانات الطقس الحية من واجهة `Open-Meteo API` لـ 81 مدينة مصرية وتخزنها في جدول `weather_readings`.
   - تعمل لمرة واحدة عند التشغيل ثم تتوقف بأمان بعد انتهاء التحميل (`Runs once & exits cleanly`).

3. **`weather_metabase_bi` (`Metabase Dashboard`)**:
   - حاوية واجهة التحليلات ولوحات البيانات التفاعلية (`Business Intelligence`).
   - مربوطة محلياً بالمنفذ (`Port 3000`) عبر: `http://localhost:3000`.
   - **الرابط العام المباشر على سيرفر الإنتاج السحابي (`Live Cloud URL`)**: [`http://72.62.92.93:3000/public/dashboard/ab63c544-c807-47da-a8c9-1490ce34f57b`](http://72.62.92.93:3000/public/dashboard/ab63c544-c807-47da-a8c9-1490ce34f57b) (يتيح للمقيّمين وأعضاء الفريق استعراض لوحات تحليلات الطقس لـ 260 مدينة مباشرة عبر الإنترنت دون تسجيل دخول).

## 🛠️ كيفية تشغيل الخدمات وإدارتها بسهولة على نظام ويندوز (`Quick Scripts`)

لقد قمنا بتوفير سكريبتات `Batch (.bat)` جاهزة لتسهيل الإدارة بضغطة زر واحدة دون الحاجة لحفظ أوامر التيرمينال:

- **`start_services.bat`**: يقوم بتشغيل كافة الخدمات (`PostgreSQL + ETL Pipeline + Metabase BI`) في الخلفية ويعرض روابط الدخول مباشرة.
- **`stop_services.bat`**: يقوم بإيقاف كافة الحاويات وتنظيف الشبكات بأمان دون فقدان البيانات المخزنة.
- **`run_pipeline_now.bat`**: يقوم بإعادة تشغيل بايبلين استخراج الطقس (`weather-etl`) لسحب أحدث قراءات الطقس الحالية فوراً وعرض السجلات (`Logs`) مباشرة في الشاشة.

## 💻 التشغيل اليدوي من سطر الأوامر (`Manual CLI Commands`)

إذا كنت تفضل تشغيل الأوامر يدوياً من هذا المجلد أو من الجذر الرئيسي للمشروع:

```bash
# تشغيل جميع الحاويات في الخلفية
docker compose up -d

# التحقق من حالة جميع الحاويات
docker compose ps

# متابعة سجلات بايبلين الطقس لحظياً
docker compose logs -f weather-etl

# إيقاف جميع الخدمات
docker compose down
```
