# 🚀 دليل تحديث السيرفر وإضافة مدن جديدة بسهولة (Server Update & Deployment Guide)
### مشروع تخرج مبادرة رواد مصر الرقمية (DEPI) — مسار Microsoft Data Engineer

هذا الدليل السريع يشرح بالخطوات العمليّة الكاملة كيفية **إضافة مدن ومحافظات جديدة** إلى منظومة الطقس، أو **تعديل كود البايثون**، ورفع التحديثات فوراً إلى السيرفر السحابي (Hostinger VPS) ليظهر التحديث مباشرة على لوحة التحكم التفاعلية في Metabase.

---

## 🗺️ أولاً: كيفية إضافة مدن جديدة محلياً على جهازك (Windows Laptop)

1. افتح مشروعك في برنامج **VS Code**.
2. افتح ملف **`api/cities.json`** الموجود في مجلد `api/`.
3. انزل إلى نهاية الملف، وأضف المدينة الجديدة بالشكل التالي (مع مراعاة وضع فاصلة `,` بعد آخر مدينة قديمة):
   ```json
   {
     "province": "Cairo",
     "city": "New Administrative Capital",
     "lat": 30.0167,
     "lon": 31.7333
   },
   {
     "province": "Matrouh",
     "city": "New Alamein City",
     "lat": 30.8167,
     "lon": 28.9500
   }
   ```
4. احفظ الملف بالضغط على `Ctrl + S`.

---

## ⚡ ثانياً: رفع التحديثات إلى السيرفر السحابي (توجد طريقتان)

يمكنك اختيار الطريقة الأسهل لك، سواء الرفع المباشر عبر التيرمنال (`SCP`) أو عبر الـ `GitHub`:

### 🥇 الطريقة الأولى: الرفع المباشر عبر التيرمنال (SCP) — الأسرع والأسهل!
هذه هي نفس الطريقة التي استخدمناها بنجاح، ولا تتطلب جيت هب:

1. افتح تيرمنال **PowerShell** في جهازك (من داخل VS Code أو ويندوز).
2. نفذ أمر الـ `scp` لنسخ المجلد بالكامل ورفعه للسيرفر السحابي:
   ```powershell
   scp -r e:\salah\salah_programing\depi_project root@72.62.92.93:/root/
   ```
3. سيطلب منك باسورد الـ Root الخاص بالسيرفر، اكتبه واضغط `Enter`.
   *(سيتم رفع الملفات والتحديثات في خلال 5 ثوانٍ فقط).*

---

### 🥈 الطريقة الثانية: الرفع عبر جيت هب (GitHub & Git Pull)
إذا كنت تفضل إدارة النسخ عبر الريبوزيتوري الرسمي على GitHub:

1. **من تيرمنال جهازك (ويندوز):** ارفع التعديلات إلى جيت هب:
   ```bash
   git add .
   git commit -m "feat: add New Administrative Capital and New Alamein cities to pipeline"
   git push origin main
   ```
2. **من تيرمنال السيرفر السحابي:** اسحب التحديثات الجديدة:
   ```bash
   cd /root/depi_project
   git pull origin main
   ```

---

## 🐳 ثالثاً: تشغيل وتحديث البايبلاين على السيرفر (Hostinger VPS)

بعد رفع الملفات للسيرفر (سواء بالطريقة الأولى أو الثانية)، افتح تيرمنال السيرفر (عبر شاشة **Hostinger Web Terminal** أو اتصال SSH) ونفذ الآتي:

1. ادخل إلى مجلد المشروع ونفذ أمر إعادة بناء وتشغيل حاوية البايثون (`--build` ضرورية لكي يشعر Docker بالتعديل):
   ```bash
   cd /root/depi_project
   docker compose up -d --build weather-etl
   ```

2. انتظر حوالي **40 ثانية** حتى يقوم البايبلاين بسحب الطقس للمدن الجديدة، ويمكنك متابعة التقدم لحظياً ومشاهدة السجلات بالأمر:
   ```bash
   docker logs -f weather_etl_pipeline
   ```
   *(اضغط `Ctrl + C` للخروج من شاشة المراقبة بعد رؤية رسالة النجاح: `Successfully loaded records in PostgreSQL`).*

---

## 📊 رابعاً: مشاهدة المدينة الجديدة على خريطة Metabase

1. افتح متصفحك على رابط لوحة التحكم: **`http://72.62.92.93:3000`**
2. افتح داشبورد الخريطة التفاعلية التي قمت بحفظها.
3. اضغط على زر **تحديث (Refresh / F5)** في المتصفح.
4. **بوم!! 💥** هتلاقي الدبوس الجديد الخاص بـ *"العاصمة الإدارية الجديدة"* أو المدينة التي أضفتها منور على الخريطة وبياناته اللحظية ظاهرة!
